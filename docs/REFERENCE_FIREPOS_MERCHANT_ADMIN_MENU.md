## 1. 文档目的与证据口径

本文完整记录当前样本商户账号可见的后台导航，作为摊伴产品设计的对标事实库。记录某项菜单，不代表摊伴一期必须实现，也不代表该能力适合直接复制。

“完整菜单”在本文中的含义是：

- 覆盖该账号当时可见的全部左侧一级菜单、分组和目标页面。
- 单独记录“应用列表”页面内的功能卡片，因为它们不是左侧二级菜单。
- 记录已经从页面或截图确认的页签、批量入口和行内操作，但不承诺穷尽所有弹窗字段。
- 权限、商户套餐和功能开关可能隐藏其他入口，因此不能据此断言平台管理员能配置的全部功能。
- 不保存登录密码、完整商户号、银行卡、身份证件、设备 SN 等敏感数据。

采集状态采用以下口径：

| 状态 | 含义 |
| --- | --- |
| 历史快照已确认 | 已在成功登录后的 DOM 菜单树中看到 |
| 截图已确认 | 已由用户提供的后台截图确认页面或操作 |
| 待复核 | 菜单层级或名称存在语义异常，需下次登录再次确认 |

## 8. 后续复核清单

下次具备有效登录条件时优先复核：

1. “设置 → 打印机管理 → 短信设置”是否确属该分组，还是历史 DOM 层级解析造成的错位。
2. “标记支付项目管理”的业务定义、字段和使用场景。
3. 不同商户套餐、权限和功能开关是否会增加或隐藏菜单。

5. 套餐商品、加料组、模板库之间的引用关系和删除约束。
6. 外卖、店内、快递、收银台四类渠道开关究竟落在分类、商品、SKU 哪些层级。
