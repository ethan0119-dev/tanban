2026-07-19 已通过正确的 `channelStore` 商户入口成功登录“码农咖啡”账号，并逐一点击打开当前账号可见的全部 88 个侧栏菜单入口；这些入口最终对应 82 个唯一 Hash 路由。本轮验证只进行只读导航，不执行新增、修改、删除、退款、打印或配置保存操作。

## 1. 文档目的与证据口径

本文完整记录当前样本商户账号可见的后台导航，作为摊伴产品设计的对标事实库。记录某项菜单，不代表摊伴一期必须实现，也不代表该能力适合直接复制。

“完整菜单”在本文中的含义是：

- 覆盖该账号本轮实时登录时可见的全部左侧一级菜单、分组和目标菜单入口。
- 单独记录“应用列表”页面内的功能卡片，因为它们不是左侧二级菜单。
- 记录已经从页面或截图确认的页签、批量入口和行内操作，但不承诺穷尽所有弹窗字段。
- 权限、商户套餐和功能开关可能隐藏其他入口，因此不能据此断言平台管理员能配置的全部功能。
- 不保存登录密码、完整商户号、银行卡、身份证件、设备 SN 等敏感数据。

采集状态采用以下口径：

| 状态 | 含义 |
| --- | --- |
| 实时登录已确认 | 已在 2026-07-19 当前登录会话中看到菜单，并实际打开对应页面 |
| 历史快照已确认 | 已在此前成功登录后的 DOM 快照中看到，但本轮尚未逐项复核 |
| 截图已确认 | 已由用户提供的后台截图确认页面或操作 |
| 前端能力超集 | 可从公开前端包确认，但可能受套餐、权限或商户类型控制，不能计入当前账号可见菜单 |
| 页面级待核验 | 菜单入口已经确认，但业务规则、字段、弹窗或写操作尚未深入核验 |

- 发送记录、充值记录。

因此，“短信设置”确实是一套通知和短信资产配置，只是当前信息架构将它显示在“打印机管理”分组下。

公开前端包还存在另一套 `#/orderremind`“订单提醒”页面，用于商户自行配置阿里云 AccessKey、签名和模板 CODE；它不是本轮当前菜单中的“短信设置”，两者不能混为一个功能。“我的短信”的签名申请还可能涉及企业资质、法人/管理员证件和授权书，应按高敏资料上传与审核流程设计。

### 6.10 标记支付项目管理

该页面用于维护收银台的自定义线下收款标记。页面提示的使用链路为：

`收银台 → 其他方式收款 → 标记收款 → 选择自定义标记收款项`

已确认字段和入口包括：新增标记支付、查看示例、排序、标记支付名称、图标、显示状态、创建时间和操作。

“标记支付”只是收银台自定义记账或付款方式标记，不是支付机构通道，也不能单独证明资金已经到账。摊伴若实现该能力，必须配置 RBAC 权限、不可抵赖操作日志及门店对账机制。

## 7. 公开前端包暴露的能力超集

前端运行时菜单由服务端按当前商户、账号权限及功能配置返回；公开包中的静态路由、条件组件和测试/候选菜单只能作为产品能力线索，不能计入码农咖啡账号实时确认的 `10 / 88 / 29` 清单。

### 7.1 商品域

- 商品支持简单/复杂规格和规格模板；售价、包装费、库存、次日置满、成本价、商品编号、重量、货号和条形码等字段主要位于 SKU / 规格组合维度。
- 支持售卖属性、加料、按日期/星期/跨日时段售卖、每单限购、虚拟销量、热销排行和单点不配送。
- 套餐区分固定套餐和分组可选套餐，包含分组可选数量、规格加价和“加料金额另算”。
- 加料组具备最少/最多可选数量、默认项和独立价格。
- 分类还存在“分类必选”能力；快速导入只导入基础信息，规格、属性等需要导入后继续配置。

### 7.2 用户与会员域

- 用户列表实际采用“会员列表”措辞，支持导入、导出、回收站、拉黑、标签、等级、来源和注册时间筛选。
- 用户详情包含基础信息、资产记录、交易统计和其他信息；资产覆盖余额、积分、成长值、优惠券和储值卡。
- 会员等级可由成长值或付费获得，并能配置权益、升级礼包、余额、积分、优惠券和有效期。
- 储值规则可赠送余额、积分、成长值、优惠券和等级；储值记录支持退款。
- 会员批量导入可能覆盖基础资料并增减积分或储值本金，属于需要审计和二次确认的高风险写操作。

### 7.3 财务与支付域

- 资产总览按微信、支付宝、余额、储值卡、现金、标记支付、自有微信/支付宝和退款等渠道拆分。
- 财务对账包含有效订单、营业总额、商户入账、微信/支付宝支付和会员储值。
- 交易详情继续拆分平台补贴、服务费、包装费、配送费、用户实付、平台抽佣和门店实得。
- 支付设置的前端能力包含随行付平台商户编号和半屏收银台等配置，证明该前端存在随行付适配能力；它不能单独证明当前商户已启用该通道、签约主体或真实资金流。

设计上必须分别保存订单应收、用户实付、支付机构实收和商户应结；“商户实际银行卡入账”应作为可导入或可回传的对账口径。若支付机构不提供结算回传或银行流水，SaaS 不能凭订单状态推定实际到账。

### 7.4 装修、数据与设置域

- 点单风格覆盖分类布局、商品单双列、库存/销量/售罄展示、加载方式、商品弹窗、堂食人数页和订单页样式。
- 导航设置具备模板、恢复默认、保存并发布；启动页支持全屏或弹窗广告、自动关闭时间和多内容元素。
- 数据域分别提供外卖、店内的订单/支付/商品指标，以及当面付渠道与支付方式分析。
- 通用设置包含未支付关单、已取消订单支付后退款、超时未接单退款、催单、自提核销、评价、库存扣减、堂食先/后结账和重复付款退款等策略。
- 商家端角色权限可细分到商品、订单、数据、财务、评价、会员、优惠券、桌位预定、酒水寄存和打印等能力。

### 7.5 当前账号不可见的静态候选入口

公开包还包含下列当前账号不可见的候选入口：

- 打印机日志 `#/printlogs`。
- 操作日志 `#/logs`。
- 商家端员工管理 `#/businessManager`；它不同于当前可见的“员工列表” `#/manager`。
- 门店管理、平台权限、多平台小程序、团购核销和更多营销应用相关入口。

静态角色权限/测试树还显示 12 个一级模块、97 个入口。其来源、版本、套餐和权限口径尚不能确定，只能确认它与当前账号的实时菜单不一致，故不并入当前菜单树。
