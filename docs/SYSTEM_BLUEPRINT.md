# 摊伴 TANBAN：移动餐饮 SaaS 系统蓝图

## 1. 产品定位

摊伴不是单一的点单小程序，也不是一套孤立的运营后台。它是一套围绕“平台—商户—顾客—设备”协同运行的多租户餐饮 SaaS，优先服务咖啡摊、夜市摊、餐车、快闪店和轻量门店。

核心价值：

1. 顾客扫码后快速自助点单并用微信支付。
2. 新订单同时进入商户工作台、微信/公众号提醒和云打印机。
3. 商品、库存、会员、营销、财务与经营数据共用同一套业务数据。
4. SaaS 平台可以统一管理商户、套餐、能力开关、小程序版本、支付与服务质量。

## 2. 对现有商户系统的实测盘点

### 首页与全局工具

- 商户概况、待办事项、实时营业额、订单数、支付人数、取消/售后、转化率。
- 外卖、堂食、快餐、当面付等渠道概览与商品 TOP10。
- 收银台、订单语音提醒、主题色、更新公告、小程序入口、商家助手、账号信息。
- 小程序二维码、聚合码、自动接单、打印小票模板。

### 外卖

- 订单处理：新订单、待配送、预订单、催单、退单/售后。
- 订单处理组：订单查询、自提订单、当面付订单、评价管理。
- 订单提醒：公众号提醒、短信提醒、语音提醒。
- 配送设置：同城配送、配送费设置。
- 打印机管理：打印模板、打印机列表。
- 快递设置：满额包邮、快递运费模板。
- 外卖设置。

### 店内

- 堂食概况。
- 订单管理：堂食订单、快餐订单。
- 桌码管理、店内设置、快餐码牌管理。
- 打印机管理：打印模板、打印机列表。
- 收银台：收银台地址、收银台设置、收银员管理。

### 商品

- 商品库：商品管理、分类管理、套餐商品、加料商品、加料组、临时商品库。
- 商品扩展：规格、属性、加料、单位、商品标签、打印标签。
- 模板：规格模板、属性模板、加料模板、商品备注库。
- 快速导入。
- 批量上架/下架、渠道开关、估清/置满、库存修改、会员价、推荐、复制、回收站和库存记录。

### 用户与会员

- 用户概况、用户列表、标签管理、用户设置。
- 会员卡等级、开卡记录、会员卡设置、会员等级订单。
- 余额流水、积分流水。
- 储值规则、储值记录、储值设置。

### 财务

- 资产总览、交易记录、财务对账。

### 店铺装修

- 装修列表、全店风格、点单风格、装修模板。
- 导航设置、启动页设置、素材管理。

### 数据

- 外卖概况：订单、支付、商品。
- 店内概况：订单、支付、商品。
- 当面付概况。

### 营销与经营应用

- 优惠券、满额立减、配送费满减、下单返红包、弹窗优惠券、弹窗广告。
- 支付有礼、店铺满赠、超级换购、商品活动、兑换码、券包。
- 收银台、餐桌预定、排队取号、酒水寄存。
- 集点返红包、抽奖活动、门店新客立减。
- 会员储值、充值消费 N 倍免单、资讯中心、积分商城、积分签到。
- 云喇叭、语音合成、活码、推广码、门店 Wi-Fi。

### 设置与组织

- 基础设置、通用设置、支付设置、标记支付项目管理。
- 短信设置、打印设置、打印机列表、页面标题。

## 3. 新系统三端信息架构

### A. SaaS 平台管理端

1. 经营总览
   - 活跃商户、在线门店、平台 GMV、MRR/ARR、续费率、退款率。
   - 套餐分布、商户增长、系统稳定性、支付与打印设备状态。
2. 商户管理
   - 入驻审核、主体认证、门店与摊位、员工、停用/恢复、风险标记。
3. 套餐计费
   - 套餐、价格、试用、订阅、续费、升级、优惠码、发票。
4. 小程序管理
   - 商户授权、模板版本、配置生成、提交审核、灰度、回滚。
5. 能力中心
   - 功能开关、套餐映射、用量限制、地区限制、灰度白名单。
6. 支付分账
   - 微信支付配置、服务商模式、退款、对账、结算、平台服务费。
7. 服务工单
   - 分类、优先级、SLA、流转、商户沟通、知识库。
8. 系统设置
   - 平台品牌、组织角色、权限、消息通道、审计日志、安全策略。

### B. 商户运营端

1. 今日概况：营业额、订单、制作队列、新客、热卖商品、设备状态。
2. 订单工作台：待接单、制作中、待取餐、已完成、退款/售后。
3. 商品库存：分类、SKU、规格、加料、库存、估清、渠道上下架。
4. 用户会员：用户、标签、等级、余额、积分、储值和复购分析。
5. 营销应用：优惠券、满减、新客立减、集点、储值、会员日。
6. 财务对账：交易、退款、渠道、日结、差异与导出。
7. 店铺装修：主题、导航、首页模块、素材、二维码与聚合码。
8. 数据分析：订单、支付、商品、时段、渠道、用户、活动效果。
9. 设备与设置：打印机、打印模板、语音提醒、员工、角色、支付。

### C. 顾客微信小程序

1. 扫码识别门店/桌码/推广来源。
2. 门店页：营业状态、公告、预计出杯时间、收藏和评价。
3. 商品：分类、搜索、推荐、规格、温度、糖度、加料和库存。
4. 购物车：数量、优惠、配送/自取方式、备注和预计时间。
5. 微信支付：创建支付单、支付结果、失败恢复和退款进度。
6. 订单：取餐码、制作状态、催单、评价、再来一单。
7. 会员：积分、余额、等级、优惠券、储值与消费记录。

## 4. 关键业务链路

```text
顾客扫码
  -> 小程序识别 tenant_id + store_id + scene
  -> 拉取门店、商品、库存、营销配置
  -> 创建订单草稿并计算价格
  -> 微信支付下单
  -> 支付回调确认订单
  -> 订单事件总线
       -> 商户工作台实时推送
       -> 微信/公众号/语音提醒
       -> 打印任务队列
       -> 库存扣减
       -> 会员积分与营销履约
       -> 数据仓库与对账
```

## 5. 核心数据模型

- `tenant`：SaaS 商户租户。
- `subscription` / `plan` / `feature_entitlement`：套餐订阅与能力授权。
- `store` / `stall_session`：门店及一次出摊营业会话。
- `staff` / `role` / `permission`：员工与 RBAC。
- `mini_program` / `mini_program_release`：小程序授权与版本。
- `category` / `product` / `sku` / `modifier_group`：商品、规格与加料。
- `inventory` / `inventory_ledger`：库存与流水。
- `order` / `order_item` / `order_status_history`：订单和状态机。
- `payment` / `refund` / `settlement`：支付、退款与结算。
- `print_device` / `print_template` / `print_job`：打印设备与任务。
- `customer` / `member` / `member_level`：顾客与会员。
- `points_ledger` / `balance_ledger`：积分与余额流水。
- `coupon` / `promotion` / `promotion_redemption`：优惠与核销。
- `notification` / `webhook_delivery`：消息与外部回调。
- `support_ticket` / `audit_log`：工单与审计。

所有经营数据必须带 `tenant_id`，门店级数据同时带 `store_id`。支付、退款、库存、余额、积分和打印任务必须使用幂等键。

## 6. 推荐工程拆分

```text
apps/
  platform-web/       SaaS 平台管理端
  merchant-web/       商户运营后台/PWA
  wechat-miniprogram/ 顾客微信小程序
services/
  api-gateway/
  identity/
  tenant-billing/
  catalog-inventory/
  order-payment/
  customer-marketing/
  device-notification/
packages/
  domain-types/
  pricing-engine/
  design-system/
  observability/
```

首版可采用模块化单体，先在一个后端中保持清晰领域边界；当订单、支付或打印吞吐需要独立扩容时再拆服务。

## 7. 上线优先级

### MVP

- 平台：商户、套餐、能力开关、基础运维。
- 商户：商品、库存、订单、打印、基础数据、员工权限。
- 顾客：扫码点单、微信支付、自取码、订单状态。

### 第二阶段

- 会员、积分、优惠券、储值、精细化营销。
- 多门店、配送、收银台、财务对账和小程序模板发布。

### 第三阶段

- 智能补货、销量预测、AI 活动建议、自动化客服、开放 API 与生态插件。

## 8. 当前仓库交付边界

当前仓库提供的是可运行、可交互、可继续扩展的三端产品原型，用于确认产品结构与核心体验；支付、微信登录、小程序发布、真实云打印、多租户鉴权和数据库持久化仍需在正式工程阶段接入。
